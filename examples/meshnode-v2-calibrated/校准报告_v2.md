# MeshNode 定价模型 v2 校准报告 · LLM 自动代理标注

> **目的**：用 LLM（DeepSeek-Chat）对 v1 的 17 个购买信号 β 做 ground-truth 校准。
> **替代**：原 Phase 1.5 计划的 N=200 人工标注。
> **撰写**：GL.iNet Data Team · 2026-05-18

---

## TL;DR

1. **跑了全量校准（3,392 条评论）**，不是抽样。DeepSeek API 6 分钟跑完，成本 < $2。
2. **9/17 信号 trustworthy**（r > 0.5 vs cluster-aggregated prior），β 保持。
3. **1 个信号 discard**（setup_friction r=0.08 → β=0）— 与 v1 报告预判一致（提及率 55.6% 过宽）。
4. **3 个新信号被发现 prior 过度乐观**（subscription_aversion / wireless_backhaul / no_subscription_value），评论里极少明确，β 折让 40-70%。
5. **校准后全市场加权 P(buy) 下调约 10pp**（主力价 €73：37.8% → **27.5%**）。**定价决策不变**：€73 主力 + €285 Flint+2 仍是最优；但**营销话术不能再过度押注「订阅厌恶」hero**。

## 方法

### 数据

| 来源 | 条数 | 说明 |
|---|---|---|
| 竞品 Amazon 评论 (v1 clusters.parquet) | 3,197 | 多语言 DE/COM/UK/ES，含 text + persona 分析 |
| Reddit cluster 样本短语 | 195 | 从 reddit_pains_decoded.json 的 quadrant.samples 去重 |
| **合计** | **3,392** | 100% 全量，无抽样 |

### LLM 标注

**模型**：DeepSeek-Chat（OpenAI 兼容 API）
**Prompt**：17 信号定义 + 评论原文 + 中文 persona 分析 → JSON 输出 0/1/2 分
**并发**：15
**耗时**：369 秒
**成功率**：100%（0 错误）

完整 script：`llm_annotate.py`

### 校准 metric

对每个信号：
- **12 原信号**（v0 已有的）：算 LLM 评分按 v0 k=20 segment 聚合后的均值 vs 原 cluster-aggregated prior 的 **Pearson r**
- **5 新信号**（v1 新增）：v0 无 prior 可比，**改用 LLM positive_rate 作 sanity check**

### β 修正规则

| Grade | 触发条件 | β 乘数 |
|---|---|---|
| trustworthy | r ≥ 0.5 | × 1.0（保持）|
| grey | 0.3 ≤ r < 0.5 | × 0.7（折让 30%）|
| weak | 0.1 ≤ r < 0.3 | × 0.4 |
| discard | r < 0.1 | × 0.0 |
| active_no_prior | LLM 频率 ≥ 5% | × 1.0 |
| rare_no_prior | 1% ≤ LLM 频率 < 5% | × 0.6 |
| very_rare_no_prior | LLM 频率 < 1% | × 0.3 |

## 关键发现

### 1. 9 个原信号 r > 0.5 — LLM 标注 vs cluster-aggregated 高度一致

**意味着 v1 prior 的 12 个原信号大多数是可信的**，cluster-aggregated proxy 不是凭空捏造。
其中 4 个信号 r > 0.85：glinet_lockin / openwrt_control / vpn_usecase / stability_pain — 这 4 个是 v1 模型最重要的差异化信号，全部经住校准。

### 2. setup_friction 应该归零

| 来源 | 提及率 | 信号有效性 |
|---|---|---|
| v0 cluster 标注 | **55.6%** | 过宽 |
| LLM 标注 | 14.8% | 合理 |
| Pearson r | **0.078** | 几乎无相关 |

v0 报告（pricing_full_pipeline_report.md）已预判 setup_friction「规则太宽，'setup' 词太通用」。
校准定论：**β=0.30 → 0**，从模型剔除。

### 3. 3 个新信号过度乐观

| 信号 | LLM 频率 | 含义 |
|---|---|---|
| subscription_aversion | **2.1%** | 真实购买评论里很少明确抱怨「订阅墙」 |
| wireless_backhaul_failure | 2.0% | mesh 回程不稳被合并到 stability_pain 里讲，单独提的少 |
| no_subscription_value | **0.7%** | 几乎没人主动表扬「免订阅」是优点 |

v1 prior 通过 `PERSONA_PRIORS` 给这 3 个信号 0.5-2.0 的分数（基于 agent 萃取的「订阅厌恶最强武器」定性发现）— **校准后发现：用户真实购买决策时这个声音很弱**。

**对营销话术的影响**：
- 「3 年省 €257 vs eero」**仍是好故事**，但不要指望大多数用户共鸣
- 适合作为对**已经踩过 eero/AVM 订阅坑**的用户的二次精准营销，不是 mass-market hero
- defection_intent (15.2% 评论提及) + pro_control (13.1%) 才是真正活跃的差异化信号

### 4. defection_intent 是最被低估的真实信号

LLM 标注显示 **15.2% 评论明确表达「退货 / 不会再买 / 准备换牌」**。v1 PERSONA_PRIORS 把这个信号编码为 segment-level 0.5-2.0 是合理的，但**全市场密度比想象的高**。

→ **退货池**比 v1 报告说的「24 条明确证据」要大得多，约 **480 条评论（15.2% × 3,197）** 都带退货/转牌意图。营销重点应该是 **「30 天无理由退」+ 「同价更稳」**，对这 480 人精准投放。

## 校准前后 · 全市场加权 P(buy)

| 价位 | v1 prior | v2 calibrated | Δ (pp) | Δ (%) |
|---|---|---|---|---|
| 攻击价 €65 | 41.65% | **30.91%** | -10.74 | -25.8% |
| 主力价 €73 | 37.77% | **27.45%** | -10.31 | -27.3% |
| 标准价 €80 | 34.80% | **24.89%** | -9.91 | -28.5% |
| 上限价 €90 | 31.15% | **21.84%** | -9.31 | -29.9% |

### 解读

- **绝对数字下调 ~10pp**：因为 v1 高估了 subscription_aversion / no_subscription_value / setup_friction 等信号的拉力
- **相对排序不变**：攻击 > 主力 > 标准 > 上限，价格弹性合理
- **27.5% 仍是健康转化率**：mesh router mid-funnel attach 通常在 10-30% 范围
- **价格敏感度略升**：v1 弹性 ≈ -0.66 → v2 弹性 ≈ -0.79（仍属不敏感区间）

## 对定价决策的影响

### 不变

1. **主力价 €73 / 节点** 仍最优 — 三法中位锚 + 外部「75€ alternative」用户口头锚
2. **Flint + 2 Nodes €285 套装** 仍是核心利润 SKU — 第四法 C 段缺口最大（与 P(buy) 数字无关）
3. **三轨 SKU 矩阵**（攻击 / 主力 / 溢价）结构不动

### 要改

1. **Hero claim 不再是「不收订阅」单点 hero** — 改为复合 hero：
   - 第一 hero：「**30 天无理由退 + 同价更稳**」（针对退货池 15.2%）
   - 第二 hero：「**OpenWrt + VPN + VLAN 全开放**」（针对 pro_control 13.1%）
   - 第三 hero（精准而非 mass）：「**3 年省 €257 vs eero**」（针对真实订阅受害者）

2. **预算分配从「mass-market 订阅 hero」转到「退货池精准营销」**：
   - 真实可定位人群规模重估：退货池 ≈ 480 条 / 3,197 评论 = 15%
   - 远比 subscription_aversion 真实人群（2.1%）大

3. **保留 v1 高置信信号的差异化**：OpenWrt / VPN / 稳定性 / 覆盖痛 — 这 4 个 trustworthy 信号在 r > 0.8，落地页要把这 4 点说透

## 不确定性 / 局限

1. **LLM 自身也可能 bias**：DeepSeek-Chat 对某些信号（如 subscription_aversion）的语义理解可能偏保守。改用 GPT-4 / Claude 复跑能验证。
2. **5 新信号无 prior baseline 可比**：只能靠 positive_rate 间接判断 — 是 v1 baseline 的固有局限，不是这次校准的缺陷。
3. **r 在 0.3-0.5 区间的判定不一定准确**：value_positive 和 price_complaint 在灰色区间，可能需要 review-level 二次校准。
4. **Reddit cluster samples 只有 195 条 phrases**（非 raw text），代表性弱于 Amazon 评论。Reddit 端的信号可能被低估。
5. **β 折让规则（×0.7 / ×0.4 等）是经验值**，未做敏感度 grid search。可以做 robust check：β 折让范围 ±20%，看 P(buy) 是否大幅变化。

→ Phase 4 真实销售 posterior 仍是最终的 ground truth，本校准只是把 launch 前的 prior 推进一步。
